¿Que debe saber un diseñador gráfico si desea ser diseñador web (Diseñador Full Stack)?

Yo personalmente creo que el punto principal que la mayoria de la gente va a mencionar al momento de ser preguntado esta pregunta es que Se debe se saber de no solo saber a con Codificar pero tambien las limitaciones y que es posible realisticamente.

Muchas personas pueden tener una buena idea de como diseñar una pagina web ya que yo pienso que despues de utilizar el internet por un buen tiempo, alguien puede notar patrones de diseño que normalmente se pueden considerar buenas practicas y como resultado crean una buena interfaz y experiencia de usuario.

Pero talvez unas personas no pueden saber que es posible hacer con los requisitos o presupuuesto dado para el proyecto. Si en teoria se puede hacer una pagina completa con animaciones, videos originales, graficos unicos y espectaculares. Pero se puede implementar todo esto con un presupuesto minimo, con poco tiempo de desarrollo y sin los equipos necesarios para crear esto?

Saber codificar y saber como se hacen las cosas en un sitio web como un diseñador grafico es una habilidad que ayuda mucho en esta linea de trabajo, ya que tiene una mejor vision en como se puede hacer el proyecto que se tiene en mente, ademas de tener expectativas mas realistas o mejorar tus expectativas en como el proyecto puede verse al fin de desarrollo.

Para resumir todo esto, cosas que tener en cuenta al momento de considerar ser una Diseñador full stack son:

  • Debe de saber como funcionar el backend del proyecto
  • No solo es diseñar como se ve el sitio web, pero tambien crearlo
  • Tener en cuentra presupuesto, audiencia en mente, tipo de contenido del sitio web, el equipo disponible, etc
  • Se tiene que tambien tener que equipo es necesario para la vision del proyecto
  • Conocer varias formas de llegar a un objetivo.
  • Saber como comunicar nuestas ideas y vision con todos los involucrados
